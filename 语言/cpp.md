模板参数<> 

对比传统实现方式的问题：

| 需求       | 传统方式      | 模板方式                 |
| -------- | --------- | -------------------- |
| 存放不同类型数据 | 要为每种类型写新类 | `SmallVector<T>`一套解决 |
| 类型安全转换   | 需要手写类型检查  | `dyn_cast<T>`自动检查    |
| 创建不同指令   | 每个指令单独函数  | `Create<T>`统一接口      |

| 符号   | 作用       | 示例                  |
| ---- | -------- | ------------------- |
| `::` | 访问作用域    | `llvm::Instruction` |
| `.`  | 访问对象成员   | `inst.getOpcode()`  |
| `->` | 通过指针访问成员 | `BB->getParent()`   |
