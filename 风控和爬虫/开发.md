### 编码规范
#### Python
- 尽量import使用package全路径名，禁止使用相对路径，也不要通过sys.path.append("../../")改变当前环境变量
- lambda一行之内解决
- 减少使用全局变量
- 返回较长列表数据建议使用yield和generator
- 使用is和is not来判断是不是None
- 不建议使用 xxx == True类似的语句
- ，：；前不加空格，后加空格
#### Java
- 不使用通配符import
- 只使用空格表示空格， 不使用Tab
- 左花括号前不换行，后换行，右花括号前换行，如果有else，不换行，否则换行
### Xposed
1. xposed_init改好入口，在project， app/src/main下创建assets， 创建xposed_init，写上xposed模块的入口包名
2. AndroidManifest加入xposed信息，application下：
	 < meta-data
		 android:name="xposedmodule"
		 android:value="true"/>
	< meta-data
		 android:name="xposeddescription"
		 android:value="描述一下xposed模块"/>
	< meta-data
		 android:name="xposedminversion"
		 android:value="54" />
3. 加入xposed依赖
	build.gradle里面，常见依赖关键字有：

| 关键字            | 含义             |
| -------------- | -------------- |
| implementation | 编译+打包进APK      |
| api            | 编译+打包进APK+对外暴露 |
| compileOnly    | 仅编译可用，不打包进APK  |
| runtimeOnly    | 只在运行时可用，不参与编译  |
	build.gradle:app
	dependencies{
		compileOnly("de.robv,android.xposed:api:82")
	}
